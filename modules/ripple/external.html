Ripple

the Fubu alternative to NuGet

Environment

  • Highly modular app
  • Team City
  • Frequent module builds published to a gallery
  • Plenty of artifacts

Requirements

  • Handling artifacts published on regular basis
  • Not changing you project during updates
  • File/Nuget Gallery feeds handling (test purposes).

Package management for .NET

  • NuGet
  • OpenWrap (from OpenRasta)
  • Ripple (from Fubu)

Ripple to the rescue

  • Two kinds of dependencies: float / fixed
  • Dependencies extracted to directories with no version!
  • Versions declared on the solution ripple.config file level
  • But still we use NuGet packages

Ripple codebase

  • Opensource: https://github.com/DarthFubuMVC/ripple
  • Part of the Fubu family
  • Very well tested
  • Fun to work with
  • We did over 5% of commits ^_^

But what it really ripple change

Csproj modification is easy:

 <ItemGroup>
    <Reference Include="Autofac, Version=3.3.0.0, Culture=neutral, PublicKeyToken=17863af14b0044da, processorArchitecture=MSIL">
      <SpecificVersion>False</SpecificVersion>
---      <HintPath>..\packages\Autofac.3.3.1\lib\net40\Autofac.dll</HintPath>
+++      <HintPath>..\packages\Autofac\lib\net40\Autofac.dll</HintPath>
    </Reference>

Also packages folder is more clear

But it has also a disadvantage: you cannot use more than one version of nuget package in one project
